# 奇语言开发指南

## 一、架构概览

### 整体流程

```
源代码 → 词法分析器 → 语法分析器(上下文关键字)
  → AST → 编译器 → 字节码(Chunk) → VM(栈式) → 执行
                    ↘ 序列化 → 编译缓存（.简令）
```

默认入口 `python3 -m qi run <文件>` 不经过格式化器。每个阶段独立、可测试、可替换。

### 执行方式

奇语言默认走**字节码 VM**：

```
CLI: python3 -m qi run 文件.qi
     ├─ 缓存命中 → 反序列化 Chunk → VM 执行
     └─ 缓存未命中 → Lexer → Parser → Compiler → 序列化缓存 → VM 执行
```

内部也保留了树遍历解释器（`use_vm=False`），仅用于对照验证。`qi run` 走直接 Compiler→VM 路径（不经 Interpreter）。

## 二、模块职责

### 记号.py（Token 类型）

**职责**：定义所有 token 类型枚举。

**为什么单独一个文件？**
- token 类型是语言的原子单位，被词法分析器、语法分析器、解释器共同使用
- 放在一起避免循环依赖

### 定义加载器.py（配置加载器）

**职责**：从 TOML 文件读取语言配置，映射为 Python 对象。

**为什么用 TOML 而不是 Python？**
- TOML 可被任何语言解析（未来编译器不需要 Python 运行时）
- 纯数据，不含代码，更安全
- 人类可读可编辑

**为什么不直接用 Python？**
- Python 配置会引入 Python 依赖，裸机编译器无法读取
- TOML 是跨语言标准格式

### 词法分析器.py（Lexer）

**职责**：源代码 → token 流。

**核心算法：`_try_split` 关键字边界切分**
- 读取连续中文序列，在关键字边界处切分
- 三个切分集合，按优先级逐位匹配：
  - `_ALWAYS_CN_TOKENS` — 始终切分（`加`、`减`、`等于`、`且`…）
  - `_STMT_START_TOKENS` — 序列开头或标点后切分（`令`、`若`、`返回`…）
  - `_TYPE_TOKENS` — 仅在 `令`/`设` 后切分（`整数`、`文本`…）
- 多字词优先匹配（`大于等于` → `再若` 不会被拆成 `再`+`若`）

**已知限制：**
- 中文标识符如果包含运算符子串会被拆分
- 已排除 `除`（防"删除"拆分）、`乘`（防"阶乘"拆分）
- 解决方案：选名时避开运算符子串，或用空格分隔

### 表达式解析器.py（ExprParser）

**职责**：关键字上下文解析 + 表达式优先级解析。

**方法**（12 个表达式优先级方法）：
- `parse_expression()` → `parse_ternary()` → `parse_logic_or()` → `parse_xor()` → `parse_logic_and()` → `parse_comparison()` → `parse_additive()` → `parse_multiplicative()` → `parse_unary()` → `parse_primary()`
- 每个方法处理一个优先级，按 Pratt 解析器惯例

**核心：`_resolve()` 上下文关键字解析**
- 根据 `_prev`（上一个已消费 token 的解析后类型）判断 IDENT 含义
- ALWAYS（关键字.py）→ 在任何位置都是关键字（的、为…）
- STMT_START → 在语句开头是关键字（令、若、返回…）
- AFTER_LET → 跟在 `令` 后是类型关键字（整数、文本…）
- EXPR_KEYWORDS / BIN_OPS / UNARY_OPS → 在表达式内部是关键字
- **注意**：词法分析器的 `_ALWAYS_CN_TOKENS`（切分用）≠ 关键字.py 的 `ALWAYS`（解析用），两者各自独立

### 语句解析器.py（StmtParser）

**继承**：`StmtParser(ExprParser)`，复用表达式解析和关键字解析。

**职责**：语句级解析 + 块解析。

**设计：缩进分块（v0.2.0+）**
- `：` 后换行 + 缩进 → 块开始（INDENT token）
- 缩进归零 → 块结束（DEDENT token）
- `：` 后同行 → 行内单语句模式（兼容旧风格）

**为什么用缩进？**
- 避免 `：`/`。` 深度计数时控制流语句消费 `。` 导致的歧义
- 与格式化器输出一致（AstFormatter 已使用缩进）
- 类似 Python，无需显式块结束标记

### 语法分析器.py

**转发层**：`from 语句解析器 import StmtParser as Parser`
——保持所有外部 import `from 语法分析器 import Parser` 兼容。

### 解释器.py（Interpreter）

**职责**：遍历 AST 执行程序。

**设计模式：**
- `execute()` 处理语句节点
- `evaluate()` 处理表达式节点
- `ReturnSignal` 异常用于函数返回

**为什么用异常做返回？**
- 函数可能在深层嵌套中调用
- 异常可以立即跳出所有嵌套层
- 比手动传递返回值更简洁

### 环境.py（Environment）

**职责**：变量、函数、结构体的作用域管理。

**设计：父链词法作用域**
- 每个作用域有 parent 指针
- 查找变量时沿链向上查找
- 函数调用创建子环境

### 关键字.py（Keywords）

**职责**：关键字分类映射（v0.1.2 新增）。

**设计：上下文相关关键字**
- `ALWAYS` — 始终关键字（的、为）
- `STMT_START` — 语句起始关键字（令、设、若）
- `AFTER_LET` — 类型关键字（整数、文本）
- `AFTER_PREV` — 依赖前驱的关键字（长度在的后面）
- `BIN_OPS` / `EXPR_KEYWORDS` — 表达式位置关键字

### 编译器.py（Compiler）

**职责**：AST → 字节码 Chunk（v0.1.2 新增）。

**遍历 AST 节点，生成字节码指令序列：**
```
BinaryOp(+, 1, 2)  →  CONSTANT 1, CONSTANT 2, ADD
```

### 虚拟机.py（VM）

**职责**：栈式字节码执行引擎（v0.1.2 新增）。

**数据结构：**
- 值栈 — 所有操作数通过栈传递
- 全局变量表 — 函数、模块、变量统一存储
- 调用帧栈 — 函数调用管理

### 字节码.py

**职责**：Chunk、ObjFunction、CallFrame 数据结构。

### 指令定义.toml / 指令加载器.py

**职责**：指令集定义（数据驱动）。与词法定义一致。

### 格式化奇.py（Formatter）

**职责**：将连写代码整理成缩进格式。

### 错误报告.py（Error Reporter）

**职责**：带源码上下文的错误格式化（v0.2.0 新增）。

**核心函数：**
- `format_error(msg, source, line, col)` — 显示错误行 + `^` 指针
- `format_error_safe(e, source)` — 自动从异常提取 line/col 并格式化

**支持异常类型：**
- `LexerError`（自带 line/col）
- `ParseError`（从 token 提取 line/col）
- `CompileError`（自带 line）
- 其他 Exception — 回退到纯文本

**入口集成：** `qi run` 的异常捕获路径使用 `format_error_safe` 显示源码上下文。

### 词法定义.toml / 语法定义.toml / 运行时定义.toml

**职责**：语言特性定义拆分为三个独立文件（v0.1.2）。
- `词法定义.toml` — 关键字、类型、运算符、标点
- `语法定义.toml` — 语句起始、参数类型
- `运行时定义.toml` — 文件后缀

## 三、关键算法

### 缩进分块（v0.2.0）

`：` 后换行 + 缩进 → 块开始。缩进归零 → 块结束。

```
令 整数 阶乘 为（整数 n）：
    若 n 等于 1 则：
        返回 1。
    否则：
        返回 n 乘 阶乘（n 减 1）。
```

词法分析器追踪缩进层级，发射 `NEWLINE`/`INDENT`/`DEDENT` token：

```python
# 词法分析器：处理 \n 后计算缩进
if self.peek() == '\n':
    self.advance()
    indent = self._calc_indent()
    # 空行/注释行跳过，有效行发射 NEWLINE + INDENT/DEDENT
    tokens.append(Token(TokenType.NEWLINE, ...))
    if indent > indent_stack[-1]:
        tokens.append(Token(TokenType.INDENT, ...))
    elif indent < indent_stack[-1]:
        tokens.append(Token(TokenType.DEDENT, ...))
```

语法分析器根据 INDENT/DEDENT 确定块边界：

```python
def _parse_block_body(self):
    self.expect(TokenType.COLON)
    if self.at(TokenType.NEWLINE):    # 缩进模式
        self.advance()
        while self.at(TokenType.INDENT):
            self.advance()  # 消费多级缩进
        stmts = []
        while not self.at(TokenType.DEDENT) and not self.at(TokenType.EOF):
            while self.at(TokenType.NEWLINE):
                self.advance()
            if self.at(TokenType.DEDENT, TokenType.EOF,
                       TokenType.ELSE, TokenType.ELIF):
                break
            stmts.append(self.parse_statement())
            if self.at(TokenType.SEMI):
                self.advance()
            elif self.at(TokenType.DOT):
                self.advance()
        if self.at(TokenType.DEDENT):
            self.advance()
        return Block(stmts)
    # 行内模式：多语句用 ；分隔，。结束
    stmts = []
    while not self.at(TokenType.EOF):
        if self.at(TokenType.SEMI):
            self.advance()
            continue
        if self.at(TokenType.ELSE, TokenType.ELIF):
            break
        stmts.append(self.parse_statement())
        if self.at(TokenType.DOT):
            self.advance()
            break
    return Block(stmts)
```

**两种模式：**
- 缩进模式：`：` 后换行，用 INDENT/DEDENT 定界块，支持多级缩进
- 行内模式：`：` 后同行，多语句用 `；` 分隔，`。` 结束
- 缩进模式下 `ELSE`/`ELIF` 不消费 DEDENT（由外层逻辑处理）

**为什么这样设计？**
- 避免 `。` 双重角色（语句结束 + 深度计数）导致的歧义
- 与格式化器输出一致（已使用缩进）
- 类似 Python，无需显式结束标记

## 四、安全模型

### 文件包含沙箱

```
包括 "./math_utils.qi"。  ✓ 允许
包括 "/etc/passwd"。       ✗ 拒绝（绝对路径）
包括 "../../etc/passwd"。  ✗ 拒绝（路径遍历）
```

**实现：**
- 拒绝绝对路径
- 相对路径基于工作目录解析
- 验证解析后的路径仍在工作目录内
- 检查文件后缀是否在允许列表

### 为什么这样设计？

- 裸机环境没有文件系统权限控制
- 语言层面的沙箱是第一道防线
- 路径遍历是最常见的安全漏洞之一
