# 版本变更记录

每个版本发布时总结，记录功能变更、修复、已知问题。

---

## v0.1.3-beta

### 新增功能
- **子命令 CLI**：`python3 -m qi run/fmt/shell`，中英文子命令均支持
- **`.简令` 编译缓存**：编译产物存于 `_qicache_/`，源码未改时直接加载
- **中文数字支持**：`十`、`二十`、`一百` 等直接在源码中使用
- **格式化 + 编译 + 执行**：`qi run` 先格式化再执行，`令整数甲为十。` 直接可跑

### Bug 修复
- 函数体内控制流语句（if/while/repeat）输出丢失
- VM 下 `读` 不存储 `_input` 变量
- `是_真值` 将 `0`、`""`、`[]` 视为真（改为假）
- `且`/`或` 短路后保留原始值而非布尔值
- `包括` 编译栈污染
- 循环栈泄漏（repeat/foreach 后栈残留）
- 解释器 output 累积（多次 run 不清空）

### 架构变更
- `qi/` 目录改为 Python 包，`__main__.py` 支持 `python3 -m qi`
- `奇.py` 与 `qi/main.py` 符号链接同步
- `_parse_func_body` / `_parse_block_body` 修复嵌套块结束判断
- `RUNTIME_ERROR_MESSAGES` 和 `_infer_type` 统一到 `环境.py`

## v0.1.2-alpha

### 新增功能
- **上下文关键字系统**：关键字可当变量名使用（`长度`、`文本库` 不再冲突）
- **字节码 VM**：栈式字节码虚拟机，支持所有核心语言特性（`交互式奇解析器.py` 默认使用）
- **指令集数据驱动**：指令定义在 `指令定义.toml` 中，与词法/语法定义风格一致
- **标准库**：基础库（数值化、文字化、取模、最大值、最小值）、文本库（长度、截取、查找、替换、转大小写、去留白）、列表库（排序、反转、去重、拥有、长度）、文件库（VFS 文件读写）

### 架构变更
- 词法分析器不再硬编码关键字，关键字判断推迟到语法分析器
- 删除死代码 `表达式解析器.py`
- 参数解析逻辑统一为 `_parse_args()`
- 配置拆分为三个 TOML + `定义加载器.py`
- 全部中文命名（目录、文件、入口点）
- 英文符号链接结构收归 `qi/` 目录

### 修复
- 上下文关键字：`令狐冲` 不再被拆为 `令` + `狐冲`
- 短路径逻辑：`且`/`或` 短路求值正确实现
- `再若` (ELIF) 语法解析支持
- `抑或` (XOR) 运算符添加入表达式解析
- 词法分析器：最长匹配分词，`水果` 不被拆分

## v0.1.1-alpha

### 新增功能
- `：`/`。` 括号式配对块结构
- `；` 连接块内多条语句
- 三元表达式：`若 condition 则 A 否则 B`
- 一元负号：`-5`、`-(1+2)`
- 类型推导：`令 X 为 10` 自动推断
- 列表元素赋值：`设 列表[j] 为 值`
- 结构体字段修改：`设 小明的年龄 为 19`
- `::` 半角注释（归一化为 `注：`）
- 文件包含安全加固（路径沙箱）
- qish 管道模式和 readline 支持

### 重构
- 源码文件全部中文命名
- 配置迁移到 TOML
- qifmt 重写为 lexer 驱动
- 表达式解析器拆分为独立模块
- STMT_STARTERS 从配置导入
- 移除无用的 `=` 检测代码
- 测试文件全部中文命名
- 文档同步更新
