# 版本变更记录

每个版本发布时总结，记录功能变更、修复、已知问题。

---

## v0.2.1-beta

### Bug 修复
- **`阶乘` 拆分修复**：从 `_ALWAYS_CN_TOKENS` 移除 `乘`，同步清理格式化器的 `_KEYWORDS`
- **全部 Bug 报表 14 项**：行内多语句、函数体语句丢弃、结构体参数验证、Include 安全、多级缩进、VM 帧上限、代码质量

### 功能改进
- **编译缓存落地**：`_cmd_run` 分离编译/执行，编译成功即写缓存；`Chunk.serialize()` 包含 `max_slot`
- **错误信息质量**：`源码/错误报告.py` 新增源码上下文格式化（错误行 + `^` 指针），集成在 `qi run` 中
- **注释重构**：通读所有源码，注释只描述事实，不描述修改动作

## v0.2.0-beta

### 重大变更
- **缩进分块**：`：` 后换行 + 缩进定义代码块，替代 `。` 深度计数
  - 词法分析器发射 NEWLINE/INDENT/DEDENT token
  - 解析器支持两种模式：缩进模式（多语句）和行内模式（单语句，兼容旧风格）
  - 格式化器同步更新，所有示例采用新风格

### 新增功能
- **Shell 多行输入**：检测 `：` 结尾行，进入 `...` 多行模式，空行结束
- **重复声明统一检查**：变量/列表/函数全部在编译时报错，不再静默覆盖
- **语法分析器拆分**：`表达式解析器.py`（ExprParser） + `语句解析器.py`（StmtParser）
  - 转发层保持 `from 语法分析器 import Parser` 兼容

### Bug 修复
- **foreach VM 栈崩溃**：`对 列表 每个 X：` 因栈未预分配导致 IndexError
  - 方案B：ObjFunction.max_slot 跟踪峰值槽位，call() 时预分配栈
- **`文本库` 拆分**：`_check_types` 标志仅在 `令`/`设` 后启用类型关键字切分
- **`截取`→`截`+`取`、`读取`→`读`+`取`**：移除 `_ALWAYS_CN_TOKENS` 中废弃的 `取`/`于`，`_STMT_START_TOKENS` 中移除 `读`
- **`qi run` 格式器破坏 AST**：`qi run` 执行路径绕过格式化器

### Bug 修复（续）
- **`阶乘` → `阶`+`乘` 拆分**：从 `_ALWAYS_CN_TOKENS` 移除 `乘`，避免拆分"阶乘"
  - 同步修复所有测试/文档/示例中的函数名
- **Bug 报表全部 14 项（P0+P1+P2）**：行内多语句截断、函数体语句丢弃、结构体参数验证、Include 安全、多级缩进、VM 帧上限、代码质量（重复关键字、TOML 同步、Node.line、死代码）

### 新增功能
- **编译缓存落地**：`_cmd_run` 分离编译/执行，编译成功即写缓存，缓存读取失败自动回退
  - `Chunk.serialize()` 序列化 `max_slot`（栈预分配），向后兼容旧格式
  - `qi run` 使用直接 Compiler→VM 路径，不再依赖 `Interpreter._last_chunk`
- **错误信息质量**：`源码/错误报告.py` 新增带源码上下文的错误格式化
  - `format_error(msg, source, line, col)` — 显示错误行 + `^` 指针
  - `format_error_safe(e, source)` — 自动提取异常中的 line/col
  - 支持 LexerError/ParseError/CompileError，集成在 `qi run` 异常捕获中

### 架构变更
- `源码/语法分析器.py`（682 行）拆分为三个文件
  - `表达式解析器.py`：关键字解析 + 12 个表达式优先级方法
  - `语句解析器.py`：语句解析 + 缩进块解析
  - `语法分析器.py`：转发层
- `源码/错误报告.py`：新增模块，统一错误格式化

## v0.1.2-alpha

### 新增功能
- **上下文关键字系统**：关键字可当变量名使用（`长度`、`文本库` 不再冲突）
- **字节码 VM**：栈式字节码虚拟机，支持所有核心语言特性（`交互式奇解析器.py` 默认使用）
- **指令集数据驱动**：指令定义在 `指令定义.toml` 中，与词法/语法定义风格一致
- **标准库**：基础库（数值化、文字化、取模、最大值、最小值）、文本库（长度、截取、查找、替换、转大小写、去留白）、列表库（排序、反转、去重、拥有、长度）、文件库（VFS 文件读写）

### 架构变更
- 词法分析器不再硬编码关键字，关键字判断推迟到语法分析器
- 删除死代码 `表达式解析器.py`
- 参数解析逻辑统一为 `_parse_args()`
- 配置拆分为三个 TOML + `定义加载器.py`
- 全部中文命名（目录、文件、入口点）
- 英文符号链接结构收归 `qi/` 目录

### 修复
- 上下文关键字：`令狐冲` 不再被拆为 `令` + `狐冲`
- 短路径逻辑：`且`/`或` 短路求值正确实现
- `再若` (ELIF) 语法解析支持
- `抑或` (XOR) 运算符添加入表达式解析
- 词法分析器：最长匹配分词，`水果` 不被拆分

## v0.1.1-alpha

- `：`/`。` 括号式配对块结构

- 一元负号：`-5`、`-(1+2)`

- 文件包含安全加固（路径沙箱）
- qish 管道模式和 readline 支持

- 配置迁移到 TOML
- qifmt 重写为 lexer 驱动
- 表达式解析器拆分为独立模块
- STMT_STARTERS 从配置导入
- 移除无用的 `=` 检测代码
- 测试文件全部中文命名
- 文档同步更新
