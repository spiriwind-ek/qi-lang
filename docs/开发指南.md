# 奇语言开发指南

## 一、架构概览

### 整体流程

```
源代码 → 词法分析器 → 语法分析器 → AST → 解释器 → 执行
```

每个阶段独立、可测试、可替换。

### 为什么用树遍历解释器？

树遍历（tree-walking）是最简单的执行模型：直接遍历 AST 节点并执行。优点：
- 实现简单，调试方便
- 不需要中间表示（IR）
- 适合语言设计阶段快速验证语法

缺点：性能差。但这正是我们选择它的原因——当前目标是验证语法，不是性能。

## 二、模块职责

### 记号.py（Token 定义）

**职责**：定义所有 token 类型枚举。

**为什么单独一个文件？**
- token 类型是语言的原子单位，被词法分析器、语法分析器、解释器共同使用
- 放在一起避免循环依赖

### 配置.py（配置加载器）

**职责**：从 TOML 文件读取语言配置，映射为 Python 对象。

**为什么用 TOML 而不是 Python？**
- TOML 可被任何语言解析（未来编译器不需要 Python 运行时）
- 纯数据，不含代码，更安全
- 人类可读可编辑

**为什么不直接用 Python？**
- Python 配置会引入 Python 依赖，裸机编译器无法读取
- TOML 是跨语言标准格式

### 词法分析器.py（Lexer）

**职责**：源代码 → token 流。

**核心算法：最长匹配**
- 中文关键词最多 4 个字符
- 按长度降序匹配，避免 `再若` 被拆成 `再` + `若`
- `CN_OPERATORS`、`KEYWORDS`、`TYPES` 三个字典合并查找

**已知限制：**
- 中文标识符如果包含已知运算符子串会被拆分（如 `累加` → `累` + `加`）
- 解决方案：选名时避开运算符子串，或用空格分隔

### 语法分析器.py（Parser）

**职责**：token 流 → AST。

**设计：`：`/`。` 括号式配对**
- `_parse_block_body()` 消费 `：`，depth+1
- 遇到 `。` 时 depth-1
- depth=0 时返回 Block
- `else`/`再若` 直接 break 退出

**为什么不缩进检测？**
- 缩进不可靠（token 的 col 值不准）
- `：`/`。` 配对更明确，像括号一样无歧义

### 解释器.py（Interpreter）

**职责**：遍历 AST 执行程序。

**设计模式：**
- `execute()` 处理语句节点
- `evaluate()` 处理表达式节点
- `ReturnSignal` 异常用于函数返回

**为什么用异常做返回？**
- 函数可能在深层嵌套中调用
- 异常可以立即跳出所有嵌套层
- 比手动传递返回值更简洁

### 环境.py（Environment）

**职责**：变量、函数、结构体的作用域管理。

**设计：父链词法作用域**
- 每个作用域有 parent 指针
- 查找变量时沿链向上查找
- 函数调用创建子环境

### 格式化奇.py（Formatter）

**职责**：将连写代码整理成缩进格式。

**为什么用 lexer 而不是正则？**
- 正则会拆分复合标识符（如 `学生小明`）
- lexer 和解析器使用相同的分词逻辑，保证一致性

### 语言配置.toml

**职责**：集中存储所有语言特性定义。

**结构：**
- 关键字（`令`→`LET`）
- 类型名（`整数`→`TYPE_INT`）
- 中文运算符（`加`→`ADD`）
- ASCII 运算符（`+`→`ADD`）
- 标点映射（`。`→`DOT`）
- 文件后缀（`.qi`/`.奇`）

**为什么这样组织？**
- 修改语言特性只需编辑 TOML，不动代码
- TOML 可被编译器直接读取
- 所有映射集中管理，避免散落

## 三、关键算法

### `：`/`。` 括号式配对

```python
def _parse_block_body(self):
    self.expect(TokenType.COLON)  # 消费 ：
    depth = 1
    stmts = []
    while depth > 0:
        if self.at(TokenType.ELSE, TokenType.ELIF):
            break  # else/再若 终止当前块
        if self.at(TokenType.COLON):
            depth += 1  # 嵌套块
            self.advance()
        stmts.append(self.parse_statement())
        if self.at(TokenType.SEMI):
            self.advance()  # ；连接同级
        elif self.at(TokenType.DOT):
            depth -= 1  # 关闭块
            if depth == 0:
                self.advance()
    return Block(stmts)
```

**为什么这样设计？**
- 像括号一样配对，无歧义
- 嵌套通过 depth 计数处理
- `else`/`再若` 是块的自然边界，不需要额外 `。`

### 函数体终止

函数体也用 `：`/`。` 配对。嵌套块用 `。。`：

```
令 整数 阶 为（整数 n）：若 n 等于 1 则：返回 1。否则：返回 n 乘 阶（n 减 1）。
```

第一个 `。` 关闭 if 的 then 块，第二个 `。` 关闭 else 块，第三个 `。` 关闭函数体。

## 四、安全模型

### 文件包含沙箱

```
包括 "./math_utils.qi"。  ✓ 允许
包括 "/etc/passwd"。       ✗ 拒绝（绝对路径）
包括 "../../etc/passwd"。  ✗ 拒绝（路径遍历）
```

**实现：**
- 拒绝绝对路径
- 相对路径基于工作目录解析
- 验证解析后的路径仍在工作目录内
- 检查文件后缀是否在允许列表

### 为什么这样设计？

- 裸机环境没有文件系统权限控制
- 语言层面的沙箱是第一道防线
- 路径遍历是最常见的安全漏洞之一
